1. Запусти все тесты в проекте стандартной командой Pytest.
pytest


2. Выполни тесты только из одного указанного файла. 
pytest test_DZ09102025.py 


3. Запусти один конкретный тест по его имени. 
pytest -k "test_class_name" 
или pytest -k test_class_name 


4. Выполни тесты с подробным выводом в консоль.
pytest -v 


5. Запусти тесты так, чтобы отображались все сообщения print(). 
pytest -s


6. Проверь, как меняется формат ошибок при использовании разных вариантов параметра --tb.
pytest --tb=short - короткий формат

pytest --tb=long - длинный формат (по умолчанию)

pytest --tb=line - только строки с ошибками

pytest --tb=no - минимальный вывод

pytest --tb=native - max отображение всех внутренних вызовов


7. Запусти тесты с отображением логов уровня INFO в консоли. pytest --log-cli-level=INFO
    

8. Сохрани все логи выполнения тестов в файл. pytest --log-file=test_log1.log 
Максимум информации:
pytest --log-file=logs.log --log-file-level=DEBUG

Только проблемы: 
pytest --log-file=logs.log  - по умолчанию WARNING

Для анализа выполнения:
pytest --log-file=logs.log --log-file-level=INFO


9. Пометь тесты меткой @pytest.mark.smoke и запусти только их. 
pytest -m smoke


10. Запусти все тесты, кроме отмеченных меткой smoke.
pytest -m "not smoke"


11. Выполни тесты параллельно в нескольких потоках. pytest -n 2
С начало устанавливаем библиотеку, если ранее не устанавливали: pip install pytest-xdist


12. Сформируй отчёт о тестах в формате JUnit XML.
pytest --junitxml=report4.xml


13. Сгенерируй HTML-отчёт с результатами выполнения тестов.
pytest --html=report5.html


14. Проведи запуск тестов с ограничением количества допустимых ошибок.
pytest --maxfail=2                                                                   


15. Проверь возможность комбинированного запуска тестов с фильтрацией по имени и метке.
pytest -k "xpath" -m smoke 


16. Выполни тесты в тихом режиме без лишней информации в консоли.
pytest -q                                                    


17. Запусти тесты с максимальным уровнем детализации (-vv).
pytest -vv


18. Проверь создание отчёта с логами и результатами тестов в одном HTML-файле.
pytest --html=log_report7.html --self-contained-html

19. Отключи захват вывода полностью и проанализируй, как отображаются сообщения в консоли.
pytest -s

Анализ: 
1. Захват вывода отключен.
2. print() сообщения видны в консоли: test_DZ09102025.py Ввести текст в поле 'name' Какой-то текст!
3. Если есть ошибки, то вывело бы и ошибки.
4. Видно, что происходит внутри кода, видно какой текст должен выдать, при той или иной ошибке.

20. Настрой запуск тестов с одновременным выводом логов и сохранением их в файл.
pytest -v -s --log-cli-level=INFO --log-file-level=DEBUG --html=final_report.html --self-contained-html
